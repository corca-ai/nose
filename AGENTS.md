## Documentation Guide

Documentation is everything. Search and read relevant docs before you touch anything. Update relevant docs after you touch anything. Follow these principles:

- Treat docs like code: reduce duplications, reveal intentions, and keep them clear and simple.
- Prefer small focused docs with Single Reponsibility Principle over long mixed docs.
- Link related docs, just like Wiki.
- Use `awiki` CLI to keep docs up to date. Use `<PROJECT_ROOT>/docs` as `--root`.
- Keep `<PROJECT_ROOT>/README.md` minimal and point to `<PROJECT_ROOT>/AGENTS.md`.
- Keep `<PROJECT_ROOT>/AGENTS.md` minimal and link to detailed docs in `<PROJECT_ROOT>/docs/`.
- Use `<PROJECT_ROOT>/docs/home.md` as the entry point of the entire project documentation.
- DO NOT MODIFY THIS DOCUMENT.

Note: `<PROJECT_ROOT>/CLAUDE.md` is a symlink to `<PROJECT_ROOT>/AGENTS.md` since Claude Code doesn't follow `AGENTS.md` standard.
